# ☁ 万物手札 - 云端同步使用指南

## 快速开始

### 第一步：获取 GitHub Token

1. 访问 [GitHub Personal Access Tokens](https://github.com/settings/tokens)
2. 点击 **"Generate new token (classic)"**
3. 填写描述（如：万物手札同步）
4. **勾选 `gist` 权限**（只需这一个权限）
5. 点击页面底部的 **"Generate token"**
6. **立即复制 Token**（格式如 `ghp_xxxxxxxxxxxxxxxxxxxx`）
   - ⚠️ Token 只会显示一次，离开页面后无法再查看

### 第二步：配置同步

1. 打开万物手札 → 点击底部 **"我"**
2. 在 **"云端同步"** 分组中点击 **"同步设置"**
3. 粘贴你的 GitHub Token
4. 设置一个**加密密码**（至少 6 位）
   - 🔐 这个密码用于加密你的数据
   - 云端（GitHub）无法解密你的数据
   - ⚠️ **忘记密码 = 数据无法恢复**，请务必牢记
5. 点击 **"测试连接"** 验证 Token 是否有效
6. 点击 **"保存设置"**

### 第三步：开始同步

#### 首次上传（本地 → 云端）

1. 点击 **"上传到云端"**
2. 输入你的加密密码
3. 等待上传完成
4. ✅ 状态显示 **"刚刚同步"**

#### 换设备下载（云端 → 本地）

1. 在新设备上打开万物手札
2. 配置相同的 GitHub Token 和加密密码
3. 点击 **"从云端下载"**
4. 输入加密密码
5. ✅ 数据自动恢复

#### 双向同步（推荐）

- 点击 **"双向同步"**
- 自动合并两端数据
- 如果同一记录在两端都有修改，**较新的版本优先**

## 安全说明

### 零知识加密

```
你的浏览器 → AES-256-GCM 加密 → GitHub Gist（密文）
              ↑
         加密密码（仅你知晓）
```

- ✅ 数据在**本地浏览器**加密
- ✅ 云端（GitHub）只存储密文
- ✅ 服务端无法查看你的数据内容
- ✅ 即使 GitHub 被入侵，你的数据也是安全的

### 加密算法

- **密钥派生**: PBKDF2-SHA256（100,000 次迭代）
- **数据加密**: AES-256-GCM
- **随机数**: 每次加密使用不同的 Salt 和 IV

## 常见问题

### Q: 同步会覆盖我的数据吗？

A: 不会。双向同步采用**智能合并策略**：
- 本地独有的记录 → 保留
- 云端独有的记录 → 下载
- 同一记录两端都有修改 → 取较新的版本

### Q: 我可以删除云端数据吗？

A: 可以。访问你的 GitHub Gist 列表，找到名为 **"📦 万物手札 云端同步数据（已加密）"** 的 Gist，删除即可。

### Q: 换设备后如何恢复数据？

A: 只需三步：
1. 新设备上打开万物手札
2. 配置相同的 GitHub Token 和加密密码
3. 点击 **"从云端下载"** → 输入密码 → 完成

### Q: 加密密码忘记了怎么办？

A: 无法恢复。由于是零知识加密，没有密码就无法解密数据。建议：
- 使用密码管理器（如 Bitwarden、1Password）
- 或记录在安全的地方

### Q: 同步失败怎么办？

A: 检查以下几点：
1. GitHub Token 是否有效（重新生成一个）
2. 网络连接是否正常
3. 加密密码是否正确
4. 浏览器控制台是否有错误信息（F12 查看）

## 技术细节

### 数据存储结构

```json
{
  "items": [...],  // 你的物品数据（加密后）
  "metadata": {
    "app": "universal_journal",
    "version": "2.0",
    "syncedAt": "2025-04-26T12:00:00Z",
    "recordCount": 156
  }
}
```

### 同步文件

- 文件名：`journal.sync.json`
- 存储位置：GitHub Gist（私密）
- 可见性：仅你本人可见（private gist）

### 自动同步

- 启动时自动同步（如果已配置）
- 修改后建议手动点击 **"双向同步"**
- 未来版本将支持后台自动同步

## 限制与注意事项

### GitHub API 限制

- **速率限制**: 5000 次/小时（已足够日常使用）
- **Gist 数量**: 无限制
- **单文件大小**: 最大 100MB（手札数据远小于此）

### 建议

1. **定期备份**: 即使有云端同步，也建议定期导出 JSON 备份
2. **密码管理**: 使用密码管理器保存加密密码
3. **网络环境**: 同步时需要能访问 GitHub API
