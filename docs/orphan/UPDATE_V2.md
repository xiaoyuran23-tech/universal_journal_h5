## 1️⃣ IndexedDB 存储

### 为什么需要？

**问题：** localStorage 只有 5MB 限制
- 每条记录约 1KB
- 每张 Base64 照片约 100-500KB
- 50 张照片就满了！

**解决：** IndexedDB 提供 50MB+ 存储空间
- 支持存储数千条记录
- 支持数百张照片
- 支持索引查询（按分类/日期/收藏）

### 技术实现

```javascript
// 数据库结构
universal_journal_db/
├── items           # 物品数据
│   ├── keyPath: _id
│   ├── indexes: createdAt, updatedAt, category, isFavorite
├── sync_logs       # 同步日志
│   ├── keyPath: id (autoIncrement)
│   ├── indexes: timestamp, type, success
└── metadata        # 元数据
    └── keyPath: key
```

### 自动迁移

首次启动时自动从 localStorage 迁移：
```javascript
const migration = await IDB.migrateFromLocalStorage();
// 迁移成功后保留 localStorage 作为备份
```

### API 使用

```javascript
// 保存物品
await IDB.saveItem(item);

// 批量保存
await IDB.saveItems(items);

// 获取所有
await IDB.getAllItems();

// 获取修改过的（增量同步用）
await IDB.getModifiedItems();

// 统计信息
const stats = await IDB.getStats();
// { itemCount: 156, estimatedSize: '2.3 MB', lastSync: '...' }
```

## 2️⃣ 增量同步

### 为什么需要？

**问题：** 每次同步都上传全部数据
- 浪费流量
- 浪费 API 调用次数
- 同步速度慢

**解决：** 仅上传修改过的记录
- 首次同步后标记所有记录为"已同步"
- 修改时自动标记为"_modified: true"
- 增量同步只上传修改过的记录

### 工作原理

```
本地修改                    云端
  │                         │
  ├── 用户修改记录 A ──────►│
  │   _modified: true       │
  │                         │
  ├── 用户新增记录 B ──────►│
  │   _modified: true       │
  │                         │
  ├── 点击"增量同步" ───────►│
  │                         │
  │◄───── 仅上传 A 和 B ─────┤
  │                         │
  └── 清除修改标记 ─────────►│
      _modified: false
```

### 使用方式

设置页 → 云端同步 → **⚡ 增量同步**

### 代码示例

```javascript
// 标记为已修改
await IDB.markAsModified(itemId);

// 获取所有修改
const modified = await IDB.getModifiedItems();

// 增量上传
const result = await CloudSync.uploadIncremental(password);
// { success: true, uploaded: 3, message: '...' }

// 清除标记
await IDB.clearModifiedFlags();
```

## 3️⃣ 冲突处理

### 为什么需要？

**场景：** 手机和电脑都修改了同一条记录

```
时间线：
T1: 手机修改记录 A（本地版本）
T2: 电脑修改记录 A（云端版本）
T3: 手机同步 → 冲突！
```

### 冲突检测

```javascript
mergeItems(localItems, remoteItems) {
  // 检测同一记录的 updatedAt 是否不同
  if (localTime !== remoteTime) {
    conflicts.push({
      id: item._id,
      name: item.name,
      local: localItem,
      remote: remoteItem,
      localModified: '2025-04-26T10:00:00',
      remoteModified: '2025-04-26T12:00:00'
    });
  }
}
```

### 解决策略

| 策略 | 说明 | 适用场景 |
|------|------|----------|
| **手动选择** | 弹窗让用户逐个选择 | 冲突少，需要精确控制 |
| **全部保留本地** | 本地版本覆盖云端 | 本地是最新修改 |
| **全部保留云端** | 云端版本覆盖本地 | 云端是最新修改 |
| **自动取新** | 根据时间戳自动选择 | 默认策略 |

### UI 交互

```
⚠️ 发现冲突

以下记录在云端和本地都有修改，请选择保留哪个版本：

┌─────────────────────────────────────┐
│ 📝 记录名称                          │
│ ┌─────────────┐ ┌─────────────┐   │
│ │ 📱 本地版本  │ │ ☁️ 云端版本  │   │
│ │ 10:00       │ │ 12:00       │   │
│ │ [选中]      │ │             │   │
│ └─────────────┘ └─────────────┘   │
└─────────────────────────────────────┘

[全部保留本地] [全部保留云端] [手动选择]
```

## 4️⃣ 同步日志

### 为什么需要？

**问题：** 同步失败不知道原因
- 是网络问题？
- 是密码错误？
- 是 Token 过期？

**解决：** 记录每次同步的详细信息
- 同步类型（上传/下载/双向/增量）
- 成功/失败
- 耗时
- 错误信息
- 数据量统计

### 日志结构

```javascript
{
  id: 1,
  type: 'sync_bidirectional',
  success: true,
  message: '双向同步完成，共 156 条记录，冲突 2 个',
  timestamp: '2025-04-26T12:30:00Z',
  duration: 1234,  // 毫秒
  remoteCount: 150,
  localCount: 155,
  mergedCount: 156,
  conflictCount: 2
}
```

### 查看方式

设置页 → 云端同步 → **📋 同步历史**

### UI 展示

```
📋 同步历史

✅ 🔄 双向同步
   同步完成：云端 150 条，本地 155 条
   2025-04-26 12:30  1234ms

❌ 📥 下载
   解密失败：密码错误
   2025-04-26 12:25  56ms
```

## 使用流程

### 首次使用（全新安装）

```
1. 打开 App → 自动初始化 IndexedDB
2. 设置页 → 云端同步 → 同步设置
3. 输入 GitHub Token + 加密密码
4. 点击"上传到云端"（首次完整上传）
5. ✅ 完成
```

### 日常使用

```
方案 A：手动同步
- 修改数据后 → 点击"增量同步"（快速）
- 或点击"双向同步"（完整合并）

方案 B：自动同步
- 配置后每次启动 App 自动同步
- 静默后台执行
```

### 换设备恢复

```
1. 新设备安装 App
2. 配置相同的 Token + 密码
3. 点击"从云端下载"
4. 如有冲突，选择保留版本
5. ✅ 数据恢复
```
