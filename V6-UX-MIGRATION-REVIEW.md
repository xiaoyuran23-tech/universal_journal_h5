# 九子圆桌评审 - UX 迁移方案

## 输入设计
- 来源：React/TypeScript 组件化首页
- 目标：万物手札 v6.0.0 (原生 JS + Store + Plugin)

## 核心评审结论

### 🔴 风险点
1. **分类系统与现有 Tags 冲突** - 原设计有固定 4 分类，需改为可选 Tag 筛选
2. **FeaturedCard 需要新数据字段** - 原设计有 `isFeatured`，Store 无此字段 → 改为 `updatedAt` 排序取最新
3. **暖色调与现有设计冲突** - 现有主色 `#4f46e5`，新设计 `#A88F6F` → 做成可选主题

### ✅ 迁移策略
1. 新增组件复用现有组件库基础
2. 分类筛选集成到 RecordsPlugin
3. 设计 Token 作为可选主题提供
